05 — GitOps
🔄 Fluxo GitOps Completo

Claude Code → Gitea → Coolify → Deploy automático

STEP 1 Claude code + /commit
→
STEP 2 Gitea git.zappro.site
→
STEP 3 Webhook push event
→
STEP 4 Coolify build + deploy
→
DONE Live ✅ health check
1
Criar branch de feature
Sempre trabalhe em branches separados do main. O main é protegido e só aceita PR.
$ git checkout -b feat/nome-da-feature
2
Desenvolver com Claude Code
Claude implementa, testa e commita incrementalmente. Use /commit para commits rápidos.
/commit "fix: descrição do que foi feito"
3
Code review antes do merge
5 Sonnets revisam o código em paralelo. Detecta bugs e problemas de segurança.
/code-review
4
Merge via PR — tudo em um comando
Faz commit + push + cria PR no Gitea automaticamente.
/commit-push-pr "feat: descrição completa da feature"
5
Deploy automático via Coolify
Coolify recebe o webhook do Gitea, faz build da imagem Docker e deploy com health check. Zero intervenção manual.
→ coolify.zappro.site → Application → Deployments ✅

06 — Guardrails
🛡️ Regras Imutáveis do Homelab

Definidas em /srv/ops/ai-governance/GUARDRAILS.md com chattr+i

⛔
Antes de qualquer ação em zona proibida, o agente PARA e pede confirmação explícita. Nunca execute comandos destrutivos sem snapshot ZFS antes.

🚫
zfs destroy*

🚫
terraform destroy*

🚫

🚫
ler id_rsa
